# 开题答辩 PPT（页级说明）

## 概览
本次答辩统一采用“系统载体 + 算法内核”的叙事结构。主验证场景固定为研究生《学术规范与专业写作》课程，研究重点聚焦于两个模块：图增强混合检索、多智能体闭环仲裁。整套主讲共 12 页，外加 2 页备份页。`Edge AI / StreamId / 端云协同` 不再进入主讲主线，只保留为已有工程基础说明。

## 页级说明

01. 页标题：[基于图增强检索与多智能体协同的智能教学平台设计与实现]；关键内容：开题题目、学校、答辩人、导师、日期、主验证课程《学术规范与专业写作》；可视化提示：深色封面 + 三个关键词卡片；关键结论：平台是研究载体，算法模块是研究重点。

02. 页标题：[背景与问题]；关键内容：多跳知识检索不足、多智能体推理稳定性不足、可信生成与用户体验存在权衡；补充说明为什么选择《学术规范与专业写作》做主场景；可视化提示：三张问题卡 + 一张场景说明卡；关键结论：教育写作场景能集中暴露本课题要解决的三个问题。

03. 页标题：[RQ 与研究框架]；关键内容：RQ、研究假设、核心方法、评价指标、预期趋势五列表；可视化提示：整页表格，突出 RQ1/RQ2/RQ3；关键结论：研究问题与实验指标一一对应。

04. 页标题：[系统定位与总体架构]；关键内容：核心研究模块与工程支撑模块分层，强调平台是载体、研究点是图增强检索与闭环仲裁；可视化提示：左文右图，复用架构图；关键结论：回答“做系统还是做算法”的问题。

05. 页标题：[前期工作基础]；关键内容：代码规模、已有页面、已有 GraphRAG 模块、已有多智能体原型、已有训练链路验证；可视化提示：四个代码规模卡片 + 两块能力说明；关键结论：项目不是从零开始，20 周主要做方法深化与实验验证。

06. 页标题：[相关工作与创新边界]；关键内容：对比现有工作，说明本研究的增量是教育场景下的集成创新，而不是基础算法原创；可视化提示：左右分栏，对照“最接近工作 / 本课题增量 / 不做的过度声称”；关键结论：避免任何基础算法原创化的高风险表述。

07. 页标题：[核心模块一：图增强混合检索]；关键内容：Dense Retrieval、Graph Expansion、RRF 融合、参数范围 `k ∈ [10,60]`；可视化提示：左文字右检索流程图；关键结论：RRF 用于处理图扩展与向量召回的异构排序融合问题。

08. 页标题：[核心模块二：多智能体闭环仲裁]；关键内容：Planner / Reasoner / Verifier、规则检查 + NLI 判别、最大回退轮次 `R=3`、澄清/拒答/最小结论降级；可视化提示：左文字右多智能体图；关键结论：多智能体价值来自闭环核验与停止规则，而不是代理数量。

09. 页标题：[实验设计与评估指标]；关键内容：主数据集为《学术规范与专业写作》课程语料与自建问答集；20 条 pilot、100 条可行性样本、300 条论文阶段扩展目标；检索对照组、仲裁消融组、τ 扫描实验；关键结论：每个 RQ 都有对应的实验路径与指标。

10. 页标题：[可行性、风险与降级方案]；关键内容：范围失控、时间不足、NLI 不稳定、人工评测资源不足四类风险及对应降级方案；可视化提示：左风险右闭环周次说明；关键结论：主动收窄范围后，20 周工作量紧凑但可完成。

11. 页标题：[进度安排与预期成果]；关键内容：20 周时间表、五阶段任务、最终交付物；可视化提示：五段式时间卡片 + 成果卡片；关键结论：报告与 PPT 的时间计划保持一致，成果围绕“系统、实验、论文”三条线闭环。

12. 页标题：[总结与 Q&A]；关键内容：选题聚焦、方法清晰、可行性可辩护三条总结；底部邀请老师批评指正；可视化提示：三张总结卡片 + 深色收束条；关键结论：回到统一口径并进入问答。

13. 页标题：[备份页 A：已有工程基础]；关键内容：Edge AI、StreamId、端云协同；说明这些内容属于已有工程探索，不是本次主研究问题；关键结论：用于回答“为什么系统工程基础看起来很多”的追问。

14. 页标题：[备份页 B：高频追问]；关键内容：RRF 为何仍有贡献、与 CRAG 的区别、为什么选择教育场景、为什么 20 周可完成；关键结论：为创新性和可行性追问准备统一答法。
