# 图片资源 (Images)

本目录包含项目使用的各种图片资源文件。

## 文件类型

### 项目截图
- 系统界面截图
- 功能演示图片
- 用户操作流程图
- 错误状态截图

### 用户界面图片
- 登录页面背景
- 图标和按钮素材
- 装饰性图片
- 品牌标识

### 技术图表
- 架构示意图
- 流程图
- 数据可视化图表
- 算法示意图

## 文件规范

### 命名规范
- 使用描述性英文名称
- 小写字母 + 连字符分隔
- 包含版本号（如需要）
- 示例：`login-page-v2.png`

### 格式选择
- **PNG**: 截图、图标、需要透明背景的图片
- **JPG**: 照片、复杂图像、文件大小敏感的场景
- **SVG**: 矢量图标、简单图形、需要缩放的图片
- **WebP**: 现代浏览器支持的高效格式

### 尺寸规范
- 截图：保持原始分辨率，必要时压缩
- 图标：提供多种尺寸（16x16, 32x32, 64x64, 128x128）
- 背景图：根据使用场景确定合适尺寸
- 缩略图：统一尺寸便于展示

## 使用指南

### 在文档中引用
```markdown
<!-- 示例：引用项目截图 -->
![描述文字] (./screenshot.png)
```

### 在代码中引用
```javascript
// 前端代码中
import logoImage from '@/assets/images/logo.png'

// HTML中
<img src="./assets/images/icon.png" alt="图标" />
```

### 在演示文稿中使用
- 复制到演示文稿目录
- 保持原始质量
- 提供替代文本描述

## 优化建议

### 文件大小优化
- 使用图片压缩工具
- 选择合适的质量设置
- 考虑使用现代格式（WebP, AVIF）
- 为不同设备提供不同尺寸

### 可访问性
- 为所有图片提供替代文本
- 确保足够的对比度
- 避免仅依赖颜色传达信息
- 考虑色盲用户的需求

### 版权合规
- 使用自己创建的图片
- 使用开源或免费图片
- 标注图片来源和许可
- 避免使用有版权争议的图片

## 维护说明

### 定期清理
- 删除不再使用的图片
- 更新过时的截图
- 检查链接的有效性
- 优化文件大小

### 备份策略
- 重要图片多处备份
- 保留原始高质量版本
- 记录图片的使用位置
- 建立图片资源索引
