# Keynote 输入文本（概览 + 要点）

## 概览
本次演示主题为“通用大模型大学教学平台 - 端云协同智能教学系统”，面向学术答辩委员会与技术评审专家。整体采用技术型学术信息图风格：Aged Cream（#F5F0E6）纸张纹理背景、深栗色标题、双语标注（ENGLISH TERM 中文翻译）、2D/等距技术插图与底部 KEY QUOTE 信息框。全稿共 15 页，按“研究背景与动机 -> 研究目标与贡献 -> 端云协同总体架构 -> 端侧与云端关键设计 -> 性能优化 -> GraphRAG 与 Multi-agent -> 硬件级优化与实验评估 -> 总结展望 -> Q&A”主线展开，突出 Edge-Cloud 协同在性能、成本、隐私与可扩展性上的综合优势。

## 要点
01. 页标题：[通用大模型大学教学平台]（Cover）；关键内容：LLM Teaching Platform - Edge-Cloud Collaborative Intelligent Education System、中文副标题“基于边缘AI与云端AI协同的通用化教学平台”、[作者姓名]、[学校名称]、2026；可视化提示：居中大标题 + 神经网络/云图标/边缘设备剪影淡纹理背景；关键结论：本研究聚焦端云协同智能教学平台并形成完整答辩主线。
02. 页标题：[研究背景与动机]；关键内容：高校教学痛点（抽象概念难呈现、答疑反馈慢、批改效率低、教学管理分散）+ AI技术机遇 + 端云协同必要性（性能/成本/隐私平衡）；可视化提示：左侧传统课堂挑战等距插图，右侧三组双语文本框；关键结论：传统教学模式亟需AI赋能，端云协同是可行优解。
03. 页标题：[研究目标与贡献]；关键内容：核心目标（通用化平台、端云协同、多学科扩展）与四大贡献（架构创新、边缘优化、GraphRAG、Multi-agent）；可视化提示：2x2 象限布局，每象限一类贡献并配图标；关键结论：项目价值集中在“架构 + 优化 + 检索 + 协作”四个层面。

06. 页标题：[端侧设计：任务分类与路由]；关键内容：本地任务（资源管理/学习追踪/简单问答）、云端任务（复杂推理/深度分析/多模态）、路由依据（复杂度/时延要求/隐私级别）、智能降级（云不可用时本地兜底）；可视化提示：输入节点 -> 决策菱形 -> 本地/云端双分支流程图；关键结论：智能路由实现“简单本地、复杂上云”的动态分工。
07. 页标题：[云端设计：服务架构]；关键内容：AI服务（FastAPI、流式输出、工具调用）、后端API（Go + Gin、RESTful、JWT）、数据库（MySQL 8.4+、GORM）、对象存储（MinIO）、容器化（Docker Compose）；可视化提示：中心微服务模块图 + 数据库/对象存储独立组件 + 连接箭头；关键结论：云端微服务实现模块解耦与独立扩展。
08. 页标题：[云端设计：核心功能模块]；关键内容：认证与权限（JWT + RBAC）、课程管理（CRUD/成员管理/教学日历）、作业系统（发布-提交-批改 + AI辅助评分）、资源中心（上传/分类/访问追踪）；可视化提示：2x2 功能宫格 + 锁/日历/文档/文件夹图标；关键结论：四大模块覆盖完整教学业务闭环。
