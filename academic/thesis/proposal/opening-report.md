# 毕业设计（论文）开题报告（简版）

## 一、课题名称
基于大模型的以学生为中心的智能教学平台设计与实现——以研究生专业英文写作课程为例

## 二、背景与意义（概述）
研究生专业写作的训练具有“任务周期长、反馈成本高、个体差异大”的特点。学生在逻辑组织、学术语气、证据使用、引文规范等方面的薄弱点往往呈现长期性与阶段性反复，单次作业评分难以支撑持续改进。与此同时，课程教学活动会持续产生高价值文本数据（写作稿件、修改记录、反馈意见、对话答疑），为“以学生为中心”的学习状态建模提供了基础。

大语言模型在文本理解与生成方面具备优势，但在学术写作辅导中仍面临“建议不可靠、引用不可追溯、缺少个体化”的风险。因此，本课题以研究生专业英文写作为试点，探索一种更可控、可追溯、可迭代的智能教学平台：既能提供高质量的写作反馈，也能沉淀学生长期画像，形成“数据—分析—干预—反馈”的闭环。

## 三、研究目标
1. 构建面向学生的学习档案与学习事件体系，实现写作能力的纵向追踪与可解释分析。
2. 实现写作类型感知的智能评估与反馈：针对文献综述、课程论文、学位论文、摘要等类型提供差异化评价与建议。
3. 引入检索增强生成（GraphRAG），让关键建议绑定课程知识库/范例片段并可引用溯源，降低幻觉。
4. 完成 Web 与移动端的跨端交付：共享核心与统一 API/SDK，UI 根据平台特性适配。
5. 建立可持续迭代的模型训练与评测管线，为后续 100k 规模数据的 LoRA/QLoRA 微调提供工程基础。

## 四、研究内容与技术路线
- **学生中心数据闭环**：围绕写作提交、AI 对话、学习时长等行为建立学习事件；在课程维度聚合后形成可追踪的学生画像与能力指标。
- **写作分析服务**：以“写作类型 + rubric + 结构化输出”为核心，输出维度评分、优点与改进建议，并支持教师复核与二次标注。
- **GraphRAG 知识库**：将课程讲义、写作规范、优秀范文片段等构建为可检索索引，回答中强制引用片段编号。
- **跨端工程落地**：采用 Monorepo + 共享包抽取 types 与 SDK；Web/Mobile 统一请求层与错误处理，平台差异仅体现在 UI/交互。
- **模型训练路线**：以 Qwen3 8B 为基座，进行 LoRA/QLoRA 微调，重点提升 tool/rag 能力与“教学引导风格”；配套离线评测与回归集控制迭代风险。

## 五、阶段性进展（截至 2026-02-08）
1. 完成 Monorepo（npm workspaces）与共享包拆分，实现 Web/Mobile 共享核心 types 与统一 SDK。
2. AI 服务已具备 OpenAI-compatible 调用能力，并实现 GraphRAG（含混合检索与索引热更新）、工具调用、引导式学习（guided）与写作类型感知分析等关键接口。
3. 训练侧已提供数据规范、数据准备脚本、LoRA/QLoRA 训练脚本与评测脚本；同时补齐“数据蒸馏 + smoke 链路验证”，可将 chat-style 数据蒸馏为 prompt/response 并输出困惑度等指标，用于训练前的快速自检与回归对比。
4. 输出并持续更新了“训练与部署指南”，支持本地 HF 推理服务接入、GraphRAG 索引构建，以及写作课程的接口联调示例。

### 2026-02-08 训练结果同步（阶段性验证）
为将训练链路结果同步至论文与开题材料，本项目在 2026-02-08 完成了“首次 all 训练 + 随机三组回归”双轨验证。该批结果仅用于验证训练、评测与文档同步链路可复现，不作为最终正式实验结论。

首次 all（`n=5`）结果如下：

| 指标 | 数值 |
|---|---|
| key_point_coverage | 0.9167 |
| refusal_accuracy | 0.8000 |
| response_format | 1.0000 |
| tool_call_accuracy | 0.0000 |

随机三组回归（每组 `n=6`，共 3 组）平均结果如下：

| 指标 | 平均值 |
|---|---|
| key_point_coverage | 0.7333 |
| refusal_accuracy | 0.7778 |
| response_format | 0.8333 |
| tool_call_accuracy | 0.0000 |

限制说明：随机三组数据属于合成随机样本，用于稳健性补充，不能替代真实 `style/tool/rag` 数据上的正式实验评估。

## 六、后续计划（概略）
1. 数据收集与标注：围绕写作类型与 rubric 建立高质量训练集与固定回归集。
2. 微调与对齐：分阶段训练 tool/rag/style，最终合并为多任务 adapter，并持续进行回归评测。已完成首次 all 训练链路验证，下一步转向真实数据正式实验。
3. 工程集成与灰度：将模型能力接入写作提交与反馈流程，支持教师复核与在线纠错闭环。
4. 论文写作与实验整理：完成系统实现、消融对比与案例分析，形成可复现的实验报告。

## 七、预期成果与验收要点
- 可部署的原型系统（Web + Mobile + Backend + AI 服务），支持写作提交、分析与个性化反馈。
- 可追溯的 RAG 辅导能力（引用片段编号，证据不足时能追问/拒答）。
- 训练与评测管线可复现，形成至少 1 套可用的 adapter 与离线评测报告。
- 完整毕业论文、系统文档与答辩材料。
