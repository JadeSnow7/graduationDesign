# API 变更回归清单

> 适用于每次接口变更后的回归记录，便于对齐需求与验收。

---

## 一、回归信息

- 变更日期：YYYY-MM-DD
- 变更模块：认证 / 课程 / 作业 / 测验 / 资源 / AI / 仿真 / 其他
- 影响范围：新增 / 修改 / 删除
- 负责人：姓名
- 关联文档：`docs/api/README.md` / 模块接口文档 / `swagger.yaml`
- 关联需求/任务：Issue/PR/需求编号

---

## 二、接口回归结果

| 接口 | 变更类型 | 用例覆盖 | 结果 | 备注 |
|------|----------|----------|------|------|
| /api/v1/xxx | 修改 | 正向 + 反向 | ✅/❌ |  |

---

## 三、关键检查点（必填）

- 契约一致性：请求字段/响应结构与文档一致
- 权限与鉴权：401/403 场景验证通过
- 错误码与提示：与规范一致
- 幂等性：重复提交行为明确
- 兼容性：客户端（Web/Mobile）未破坏

---

## 四、结论

- 是否通过：✅/❌
- 风险说明：若存在未覆盖项或暂缓问题，说明原因与补测计划。
