# 毕业论文结构设计：融合大语言模型与知识图谱的教学辅助平台

**日期**：2026-04-17  
**论文题目**：融合大语言模型与知识图谱的教学辅助平台设计——以《学术规范和论文写作》为例  
**核心调整**：将主案例由"电磁场（双案例并列）"切换为"学术规范与论文写作（单一主线）"

---

## 核心论点

> 学术写作规范可以被形式化为知识图谱；通过 GraphRAG 将 LLM 的反馈与图谱节点绑定，可以实现"有规范依据、可溯源、可验证"的写作辅助，而非凭空建议。

三个技术贡献：
1. **KG 的设计与构建**——对学术写作规范的形式化建模
2. **GraphRAG 检索绑定机制**——回答强制对齐规范节点
3. **反馈质量的可评测性**——溯源率、采纳率等可量化指标

---

### 第3章 学术规范知识图谱设计与构建（新章，核心贡献）

**3.1 领域分析与建模范围**

梳理《学术规范与论文写作》课程中需要形式化的规范维度，**按可形式化程度分层**：

**强形式化层（本文核心建模范围）**：
- 引用格式（APA/GB/T 7714 等）——规则明确，可规则化检验
- 章节结构（IMRAD 模型）——结构固定，可程序化识别
- 段落功能分类（背景、方法、结果、讨论）——可用分类模型辅助标注

**弱形式化层（KG 提供示例和引导，不作为硬约束评测）**：
- 论证组织（声明—证据—推理链）——结构可描述，但识别有模糊性
- 学术诚信约束（引用完整性、paraphrase 规范）——可部分规则化

**不纳入形式化（KG 范围之外，在第6章局限性中说明）**：
- 学术语气（hedging、主动/被动语态规范）——主观性强，难以形式化校验

**3.2 本体设计**

**KG 中的静态节点**（构建时入库，持久存储）：
- `规范条款`：具体可引用的规范要求
- `示例片段`：课程讲义中的正例/反例（与规范条款绑定）
- `违例模式`：常见错误类型的抽象描述
- `修改建议`：与违例模式对应的修改策略
- `评价维度`：Rubric 中的评分维度

**运行时对象**（不入 KG，检索时临时构建）：
- `待分析片段`：学生提交文本的切分结果，用于检索种子，不存入图谱

关系类型（仅作用于静态节点之间）：
- `对应修改` (违例模式 → 修改建议)
- `属于` (规范条款 → 评价维度)
- `前置规范` (规范条款 → 规范条款，用于有依赖关系的规范)
- `示例于` (示例片段 → 规范条款)
- `实例化` (违例模式 → 规范条款，表示该违例模式违反了哪条规范)

**3.3 构建流程（B+C 路线）**

- **C路线（基础层）**：以 ScholarlyAO 或 AO 为基础本体，映射到 HUST 学位论文写作规范，保留通用节点，补充 HUST 特有规范节点
- **B路线（扩展层）**：使用 LLM 从课程讲义、规范文档、历年范文中半自动抽取实体与关系；设计抽取提示词（含节点类型约束）；人工审校抽取结果

**3.4 知识图谱质量评估**

两层评测，方法不混用：
- **标注员一致性**：两名标注员独立对同一批关系三元组打标，用 Cohen's κ 衡量一致性（κ > 0.7 为可接受）
- **抽取效果**：以人工标注结果为金标准，计算 LLM 半自动抽取的 triple-level Precision / Recall / F1
- **节点覆盖率**：KG 中规范条款节点与课程讲义中可识别规范条目的对齐比例
- **维度完整性**：是否覆盖课程考核的全部评价维度（逐维度核查）

### 第4章 系统设计与实现

架构保留三层（React + Go + FastAPI），叙事重心转为"KG 如何驱动各模块"。

**4.1 技能系统与对话编排**（保留现有内容，补充写作场景的技能设计）
- `writing_feedback_rag`：结构化写作反馈，强制引用 KG 节点
- `writing_guided`：引导式写作改进，分步推进
- `academic_integrity_check`：学术诚信约束内置于技能提示

**4.2 GraphRAG 检索绑定与节点校验闭环**（扩写，强调"真正可验证"而非"格式上带编号"）

检索绑定四步流程：
1. **候选节点检索**：以学生文本的待分析片段为查询，混合检索（向量 + 关键词）得到候选 KG 节点列表
2. **图扩展**：从候选节点出发沿图边扩展 1-2 跳，补充前置规范和相关示例
3. **反馈生成**：将扩展后的节点注入系统提示，要求模型按 `[REF:节点ID]` 格式标注每条反馈的来源
4. **绑定校验**：生成完成后，程序自动校验每个 `[REF:X]` 引用——节点 X 是否存在于 KG、余弦相关性是否达到阈值（> 0.6）；不合格的引用触发重排（换候选节点）或标记为"低置信度"

> 这一闭环确保"可追溯"是系统约束，而非仅靠提示词约定。

- 知识库更新流程：新增规范文档如何增量更新图索引

**4.3 Rubric 驱动的反馈生成**
- 以 KG 中的`评价维度`节点作为反馈结构的锚点
- 输出格式：维度名称 → 问题定位 → 规范依据（KG节点）→ 修改建议

**4.4 规范检查工具**
- 引用格式检查工具（正则 + 规则引擎，覆盖 APA/GB/T 7714）
- 段落结构检查工具（识别段落功能标记的缺失）
- 工具结果回注到对话上下文

**4.5 学习画像与 KG 节点对齐**
- 薄弱点不再是自由文本，而是与 KG 节点绑定（例："该学生在`引用格式.APA.二次文献`上反复出错"）
- 支持按 KG 维度聚合学生群体画像，辅助教师发现班级共性问题

**4.6 训练与评测管线**（保留，补充写作场景样本类型）
- 新增样本类型：`writing_feedback`（含 KG 节点引用约束）、`norm_check`

### 第5章 实验与评估

三条评测线：

| 评测对象 | 核心指标 | 方法 |
|---------|---------|------|
| KG 质量 | 节点覆盖率、关系准确率（Cohen's κ）、维度完整性 | 人工标注对比 |
| 反馈溯源性 | KG引用命中率（反馈中引用的规范节点是否存在且相关） | 自动检查 + 人工抽样 |
| 反馈有效性 | 学生修改采纳率、修改前后 Rubric 得分变化 | 小规模用户实验（10-20人） |

**对比实验设计（控制变量说明）**：

三组方案除检索机制外，其余条件保持一致（相同的提示词模板结构、相同的输出格式约束、相同的规则检查工具）：

| 方案 | 检索机制 | KG绑定校验 |
|------|---------|-----------|
| 基线 A：纯 LLM | 无检索 | 无 |
| 基线 B：传统 RAG | 向量检索，无图结构 | 无 |
| 本文方案 | GraphRAG + 图扩展 | 有（4.2校验闭环） |

> 控制变量目的：确保指标提升来自检索机制与绑定校验本身，而非因为输出更规整。

**消融实验**：去掉图扩展（仅向量检索，但保留 KG 绑定校验）vs 完整 GraphRAG，对比引用命中率差异，验证图扩展的独立贡献。

### 第6章 结论

**总结**：
- KG 建模覆盖范围与局限：哪类规范适合形式化（格式、结构），哪类不适合（学术语气的细微差别）
- GraphRAG 绑定机制的泛化性：是否可迁移到其他规范性课程

**展望**：
- KG 自动更新（新规范文档发布后的增量维护）
- 多语言规范支持（中英文写作规范的统一图谱）
